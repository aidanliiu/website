Algorithms Covered

From classical ciphers to quantum-resistant schemes

This research examined five key cryptographic algorithms, tracing the evolution from historical ciphers to the lattice-based constructions being standardized by NIST for the post-quantum era.

Symmetric

AES

Advanced Encryption Standard. 128-bit block, 128/192/256-bit keys, 10-14 rounds. Quantum-resistant at AES-256; Grover's algorithm halves effective key length, reducing AES-128 to ~64-bit security.

Broken by Shor's

RSA

Asymmetric encryption based on integer factorization hardness. Shor's algorithm solves this in polynomial time on a quantum computer, making RSA completely insecure in the post-quantum era.

Post-Quantum Safe

Kyber KEM

Lattice-based Key Encapsulation Mechanism. Security rests on the Module-LWE problem, believed hard for both classical and quantum adversaries. Selected by NIST for standardization in 2022.

Historical / Deprecated

DES

Data Encryption Standard (1977). 56-bit key, 64-bit block. Brute-forced in 22 hours in 1999. Studied here as a historical baseline and to motivate the transition to AES and beyond.

Classical Baseline

Hash Functions

SHA-256 and SHA-3 for data integrity. SHA-3 uses the Keccak sponge construction. Both offer strong resistance under Grover's, requiring roughly double key length for equivalent post-quantum security.

NIST Standard

CRYSTALS-Dilithium

Lattice-based digital signature scheme. Paired with Kyber in the NIST PQC suite. Based on Module-LWE and Module-SIS hardness problems, designed for practical signature sizes and verification speed.

Quantum Threat Model

Two algorithms that define the threat

Quantum computing doesn't threaten all cryptography equally. The impact depends on which quantum algorithm applies to the underlying mathematical problem.

⚡

Shor's Algorithm

Developed by Peter Shor in 1994. Solves integer factorization and discrete logarithm in polynomial time on a quantum computer. Completely breaks RSA, Diffie-Hellman, and elliptic curve cryptography (ECC) once sufficiently large quantum computers exist.

Breaks RSA and ECC entirely
🔑

Grover's Algorithm

Provides a quadratic speedup for unstructured search. Effectively halves the bit-security of symmetric ciphers: AES-128 drops to approximately 64-bit effective security, while AES-256 remains strong at ~128 bits. Mitigated by simply using larger keys.

Mitigated by doubling key size

Mathematical Foundation

Why lattices resist quantum attacks

The Learning With Errors (LWE) Problem

A lattice is a regular grid of points in n-dimensional space, defined by a set of basis vectors. The visualization shows a 2D lattice with an observed vector (blue) that has been perturbed by a small error.

b = As + e (mod q)

Given public matrix A and observed vector b, recover secret s knowing only that error e is small. Even for quantum computers, this appears computationally infeasible in high dimensions.

Kyber uses the Module-LWE (MLWE) variant, which generalizes LWE to module lattices - balancing security and performance for practical key sizes without sacrificing the underlying hardness guarantees.

Protocol Design

Kyber KEM: key exchange step by step

Kyber is a Key Encapsulation Mechanism. Alice generates a key pair and shares her public key. Bob uses it to encapsulate a shared secret and sends back only the ciphertext. Alice decapsulates with her private key. Neither party ever transmits the secret directly.

Alice (Recipient)
1. Key Generation Generate public key pk and private key sk using a random seed and Module-LWE lattice structure
3. Decapsulation Use private key sk to recover shared secret K from received ciphertext c
Result K = shared symmetric key, ready for session encryption
PUBLIC KEY pk
CIPHERTEXT c
Bob (Sender)
2. Encapsulation Use Alice's pk to generate ciphertext c and a shared secret K, derived from a random value
Result K = same shared symmetric key, independently derived
Implementation

Libraries used in practice

Three open-source cryptographic libraries were used to implement and test algorithms hands-on. Each occupies a different layer of the cryptography stack.

OpenSSL is the industry-standard cryptography toolkit, supporting AES, DES, RSA, ECC, and major hash functions. Used here to implement symmetric AES and DES encryption, demonstrating the encryption-decryption pipeline and comparing security characteristics.

# Encrypt a file with AES-256-CBC
openssl enc -aes-256-cbc \
  -in plaintext.txt \
  -out encrypted.bin \
  -k "my-secret-key"

# Decrypt back to plaintext
openssl enc -d -aes-256-cbc \
  -in encrypted.bin \
  -out decrypted.txt \
  -k "my-secret-key"

# Verify round-trip integrity
diff plaintext.txt decrypted.txt
# "No differences detected" -- successful implementation

Libsodium (via Python binding PyNaCl) provides a high-level API for secure cryptographic operations. We built a Student-University secure data transmission protocol using ChaCha20-Poly1305 authenticated encryption and Ed25519 digital signatures for message authenticity.

from nacl.public import PrivateKey, Box
from nacl.signing import SigningKey

# Curve25519 key generation for both parties
alice_key = PrivateKey.generate()
bob_key   = PrivateKey.generate()

# Authenticated encryption: Alice sends to Bob
box = Box(alice_key, bob_key.public_key)
encrypted = box.encrypt(b"Transcript: GPA 3.97")
decrypted = box.decrypt(encrypted)

# Ed25519 signature for non-repudiation
signing_key = SigningKey.generate()
signed_msg  = signing_key.sign(encrypted)
signing_key.verify_key.verify(signed_msg)
# Verified -- authentic, unmodified message

Liboqs (Open Quantum Safe) provides production-ready implementations of NIST-selected post-quantum algorithms. Used here to implement the full Kyber-768 KEM pipeline: key generation, encapsulation, and decapsulation, with correctness verified by comparing the independently-derived shared secrets.

import oqs

# Initialize Kyber-768 (NIST security level 3)
with oqs.KeyEncapsulation("Kyber768") as alice:

    # Alice: generate key pair
    public_key = alice.generate_keypair()

    # Bob: encapsulate a shared secret using Alice's public key
    with oqs.KeyEncapsulation("Kyber768") as bob:
        ciphertext, K_bob = bob.encap_secret(public_key)

    # Alice: decapsulate to recover the same shared secret
    K_alice = alice.decap_secret(ciphertext)

    assert K_alice == K_bob
    print(f"Shared key: {K_alice.hex()[:32]}...")
    # Key exchange successful -- quantum-resistant session key established
Key Findings

What this research revealed

🔐

Lattice hardness holds against classical and quantum adversaries

Unlike RSA and ECC, lattice-based schemes resist both classical and quantum attacks. The SVP and LWE problems have no known efficient algorithms on either hardware class - a uniquely strong security foundation.

⏱

Standardization rigor takes years for a reason

The NIST PQC process ran from 2016 to 2022 with rounds of public cryptanalysis. Multiple candidate algorithms were broken or weakened during review. That scrutiny is exactly what makes a standard trustworthy.

⚡

Kyber achieves competitive performance

Despite larger key sizes than RSA, Kyber uses NTT-based polynomial multiplication over module lattices, enabling fast key generation and encapsulation. In benchmarks, Kyber outperforms RSA on both key-gen and decapsulation speed.

🌐

"Harvest now, decrypt later" is a present threat

Adversaries can intercept and store encrypted traffic today, waiting until quantum hardware matures to decrypt it. Systems protecting long-lived sensitive data need to begin migrating to PQC algorithms now - not when quantum computers arrive.
